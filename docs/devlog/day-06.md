## Decisions
- **ADR-0006: the corpus is a manifest, not a crawl.** Whatever is in the corpus is what an agent will believe and cite; whatever is missing it will answer from the model's priors instead of from this team's decisions. A crawl of "everything minus an exclusion list" grows silently — add a directory and the context layer starts grounding agents in whatever landed there, with nobody reviewing a diff. A rule list makes "what do the agents know" a code change with a test attached.
- **Generated, not scraped.** A corpus built by `curl`ing `localhost:8001/openapi.json` depends on which containers happened to be up, cannot be built in CI, and differs between machines. Importing the app and calling `app.openapi()` gives the same bytes everywhere. `sort_keys=True` makes the text stable, which makes the hash stable, which is what lets "unchanged" mean something.
- **`doc_id` is the path, not a UUID.** Level 1's promise is that every agent answer names its source. An opaque id would break that at the last inch — a citation has to be something a human can paste into an editor.
- **The ADR template is excluded on purpose.** It is a form, not a decision. Grounding an agent in it teaches it to answer questions with blanks.
- **The context layer does not ingest itself.** The corpus is Level 0's output. Whether the AI layer should read its own source is a real question, and it belongs to Day 9's knowledge graph as a decision — not to a glob as a side effect.
- **Pruning is not optional.** A document whose source has left the repo is worse than a missing one: an agent will cite it with full confidence. Every run reconciles the store against the manifest.

## What running it found
1. **Line endings would have broken reproducibility on day one.** Git reported `LF will be replaced by CRLF` on the very first commit of this work. A corpus hashed on a Windows checkout and re-hashed in Linux CI would have disagreed about every single document — every run reporting 65 updates, every embedding re-computed, and "unchanged" meaning nothing. Content is normalised to `\n` before hashing, and there is a test that says so.
2. **`encoding="utf-8"` is not optional on Windows.** Python's default text encoding here is cp1252, and the plans and devlogs are full of `→`, `·`, and `—`. Every read and write in the pipeline names its encoding explicitly.
3. **New code is invisible to CI until it is on the mypy line.** `bellwether/` would have type-checked cleanly on my machine and never been checked at all in CI, because the workflow types `tests substrate platform`. This is Day 5's execution note 10 from the other direction — same failure, opposite sign. The workflow now runs `mypy tests substrate platform bellwether`, and I reproduced that exact command locally before pushing.
